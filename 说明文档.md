# 说明文档

## 项目定位

这个项目是个人多账号 Token 管理工具。

主数据保存在 `tokens` 目录。

格式化导出保存在 `outputs/CPA` 和 `outputs/Sub2API`。

界面里支持手动授权、浏览器辅助自动授权、批量刷新、标签同步、CPA 远端查看、导入导出和上传。

## 目录说明

`main.py`
程序入口，默认启动 GUI。

`token_manager/`
核心代码目录。

`tokens/`
本地账号真实数据目录。刷新、授权、导入后的账号文件都在这里。

`outputs/CPA`
CPA 格式导出文件目录。

`outputs/Sub2API`
Sub2API 格式导出文件目录。

`ico/openai.png`
源图标。打包时会自动转成 `.ico`。

`build.py`
标准打包脚本。

`打包.py`
中文入口，内部转到 `build.py`。

`打包.bat`
Windows 一键打包脚本。

`build.spec`
打包配置。由 `build.py` 自动生成。

## 运行要求

建议 Python 3.11。

基础依赖：

```bash
pip install -r requirements.txt
pip install pyinstaller
```

## 启动

Windows：

```bash
启动.bat
```

或者：

```bash
python main.py gui
```

## 授权逻辑

手动授权：
生成授权 URL，你在浏览器里登录，复制回调 URL 回来保存。

自动授权：
程序本地监听回调地址，自动打开浏览器，等浏览器回调后自动换 token 并保存。

当前自动授权不是完整模拟网页登录，邮箱密码和验证码仍然是在浏览器里完成。

## 数据逻辑

`tokens` 是主数据。

重新授权、刷新、从 CPA 导入，都会更新 `tokens`。

`outputs` 是派生格式文件。

预览 CPA/Sub2API、上传 CPA/Sub2API、整理导出文件，都会更新 `outputs`。

## 打包

先准备图标和 spec，不执行真正打包：

```bash
python build.py --prepare-only
```

正式打包：

```bash
python build.py --clean
```

或者 Windows 直接双击：

```bash
打包.bat
```

打包脚本会做这些事：

从 `ico/openai.png` 自动生成 `build_assets/openai.ico`

自动生成 `build.spec`

调用 PyInstaller 产出 `dist/OpenAI-Token-Manager.exe`

推荐忽略的内容已经写进 `.gitignore`，包括：

`tokens/`

`outputs/`

`token_manager_config.json`

`dist/`

`build/`

## 发布到你自己的仓库

本地初始化后，不会自动推送。

你后面自己发布时，用下面这组命令，把 `<你的仓库地址>` 换成真实地址：

```bash
git add .
git commit -m "init openai-token-manager"
git remote add origin <你的仓库地址>
git branch -M main
git push -u origin main
```
